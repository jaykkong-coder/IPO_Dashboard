# IPO Dashboard

## 프로젝트 개요
KRX KIND 신규상장기업현황 API + DART 투자설명서 밸류에이션 데이터를 DB화하고 대시보드로 시각화.

- **GitHub**: https://github.com/jaykkong-coder/IPO_Dashboard
- **Streamlit Cloud**: 배포 완료
- **정적 HTML 대시보드**: `ipo_dashboard.html` (GitHub Pages 배포 가능)
- **DART API 키**: 359212a47d6222104789f5d610aa3896471f8227

## 아키텍처

```
KIND 신규상장기업현황 API → DART API (종목코드→고유번호 매핑) → 투자설명서 다운로드 → HTML 파싱 → SQLite
                                                                                                ↓
38커뮤니케이션 스크래핑 (기관경쟁률, 의무보유확약) ─────────────────────────────────→ DB 보강
KIND 공모가대비주가추이 크롤링 (상장일~1년 등락률) ─────────────────────────────────→ DB 보강
                                                                                                ↓
                                                                            Streamlit / 정적 HTML 대시보드
```

| 파일 | 역할 |
|------|------|
| `dashboard.py` | Streamlit 대시보드 (차트, 시계열, 통계) |
| `ipo_dashboard.html` | 정적 HTML 대시보드 (DB → 내장 JSON, 단독 배포 가능) |
| `ipo_extractor.py` | 투자설명서 파서 (핵심, 800줄+) |
| `pipeline.py` | 일괄 처리 (KIND → DART 종목코드 매핑 → 파싱 → DB) |
| `reparse.py` | 캐시 문서 재파싱 (네트워크 불필요) |
| `kind_scraper.py` | KRX KIND 신규상장기업현황 API 크롤러 |
| `scraper_38.py` | 38커뮤니케이션 크롤러 (기관경쟁률, 의무보유확약, 청약경쟁률) |
| `database.py` | SQLite 스키마 + upsert (주가등락률 필드 포함) |
| `dev_test.py` | 파서 개발용 테스트 스크립트 |

## 핵심 의사결정

1. **SPAC 제외**: 회사명/업종에 "스팩/SPAC/기업인수목적" 포함 시 skip
2. **합병상장 제외**: 투자설명서 앞 2000자에 "합병" 키워드 → skip
3. **정정 후 값 우선**: `reversed(tables)` 로 뒤에서부터 검색
4. **주당 평가가액 산출 내역 테이블 1순위**: 공모주식수/신주/구주/상장후주식수의 가장 정확한 소스
5. **발행제비용 테이블**: 인수수수료 + 수수료율의 가장 정확한 소스
6. **EV/EBITDA 적정시총**: EV - 순차입금 - 비지배지분 + 공모유입자금
7. **연환산/LTM = 일반기업** (추정치가 아님)
8. **종목코드 → DART 매핑**: KIND 종목코드(6자리) × 10 = DART stock_code. 회사명 매칭 실패 시 stock_code fallback
9. **혼합평가방법**: 복수 방법 사용 시 "PER+EV/EBITDA" 형태로 저장
10. **역산 금지**: 적정시총을 역산으로 구하지 않음. 투자설명서 원문 값만 사용
11. **KIND 크롤러 소스 변경** (2026-03): 기존 종합정보 > 기업개황 → 신규상장기업현황 API로 전면 교체. 액면가/공모가/공모금액/주요제품/최초상장주식수 직접 수집

## 파싱 주의사항

### 용어 변형 (같은 값, 다른 표현)
- 멀티플: 유사기업PER, 비교회사PER, 비교대상회사PER, 비교기업PER
- 단위: "45.5배", "25.72(배)", "30.91X", 별도셀 ['배', '1.80']
- 적용이익: 당기순이익, 순이익, EBITDA, 매출액, 자본총계

### 단위 변환
- 백만원, 천원, 원 → 모두 백만원으로 통일
- 발행제비용 단위 판단: 합계 < 10만 → 백만원, 10만~1억 → 천원, 1억+ → 원

### 증권사 인식
- "증권" or "금융투자" or "인터내셔날" 포함
- "증권의종류" 같은 헤더 제외

## 업데이트 방법

```bash
# 신규 회사 추가 (DART API 필요)
python3 pipeline.py

# 파서 수정 후 재파싱 (네트워크 불필요)
python3 reparse.py

# 파서 개발 중 빠른 테스트
python3 dev_test.py              # 전체 재파싱 + 검증
python3 dev_test.py 리브스메드     # 특정 회사만 테스트

# GitHub 배포
git add -A && git commit -m "update" && git push
```

## DB 스키마 주요 필드

- 밸류에이션: 평가방법, 적용멀티플, 적용이익_백만원, 적정시가총액_억원, 할인율, 공모가밴드/확정
- 주식구조: 상장후주식수, 유통가능주식수/비율, 공모주식수(신주/구주), 공모비율
- 수수료: 인수수수료총액_억원, 인수수수료율
- 수요예측: 기관경쟁률, 의무보유확약비율, 청약경쟁률_비례 (38커뮤니케이션)
- 주가등락률: 상장일시가/종가, 1개월/3개월/6개월/1년 (KIND 공모가대비주가추이)

## 현재 데이터 범위
- 2016 ~ 2026 신규상장 676건 (SPAC/합병/유증/사채/리츠/분할재상장 자동 제외)
- 파싱 성과: 적용이익 95%, 유통비율 92%, 적용멀티플 100%
- 38커뮤니케이션: 기관경쟁률 89%, 의무보유확약 91%
- KIND 주가등락률: 상장일~1년 수익률 수집 완료
